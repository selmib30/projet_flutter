◆ Apply

void LAPIN::Apply ( )
privateslot

Calcule et affiche l'image complète seuillée.

Auteur
Benjamin Albouy Kissi
Date
13/09/2010

Si le flag LAPIN::m_bApplied vaut true alors ne fait rien.

Sinon, demande le calcul du double seuillage complet à l'objet LAPIN::m_lapThreshold via la fonction QLAPINThreshold::DoThreshold(), puis affiche le résultat dans le cadre adéquat.

Active les actions d'enregistrement et passe le flag LAPIN::m_bApplied à true pour spécifier le double seuillage est réalisée (évitant ainsi de le recalculer si aucune modification n'est faite).

◆ Preview()

void LAPIN::Preview ( )
private

Calcule et affiche l'image de prévisualisation.

Auteur
Benjamin Albouy Kissi
Date
13/09/2010

Définit le flag LAPIN::m_bApplied à false pour spécifier que seule une prévisualisation est faite.

Si la case à cocher "Appliquer tout de suite" est cochée, alors il n'y a pas de prévisualisation à faire, et la fonction Preview() appelle simplement la fonction Apply() puis se termine.

Sinon, demande le calcul de la prévisualisation à l'objet LAPIN::m_lapThreshold via la fonction QLAPINThreshold::DoThreshold(), puis affiche le résultat dans le cadre adéquat.

Active les actions d'enregistrement et lance un timer de 3 secondes pour l'application des seuils sur l'image complète.

◆ VerifyModifications()

bool LAPIN::VerifyModifications ( void  )
private

Vérifie si des modifications non enregistrées sont survenues et propose l'enregistrement à l'utilisateur.

Auteur
Benjamin Albouy Kissi
Date
13/09/2010
Renvoie
true si la vérification est Ok, false sinon.

Si le flag LAPIN::m_bModified vaut true, alors propose l'enregistrement des modifications. Si l'utilisateur accepte, enregistre les modifications (appel de on_actionEnregistrer_triggered()).

L'utilisateur se voit proposer 3 solutions :

  • Oui : Enregistre les modifications (appel de on_actionEnregistrer_triggered()) puis renvoie true si l'enregistrement a fonctionné.
  • Non : N'enregistre pas les modifications et renvoie true.
  • Annuler : N'enregistre pas les modifications et renvoie false. Dans le cas où la fonction retourne false, la fonction appelante devrait alors annuler son opération.

Si le flag LAPIN::m_bModified vaut false, alors VerifyModifications() retourne true sans aucune autre action.
